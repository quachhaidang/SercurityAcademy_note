================================================================================
1. PROJECT OVERVIEW & ARCHITECTURE
================================================================================

PROJECT PURPOSE:
- Digitalize & secure student transcripts and certificates using SHA-256 hashing
  and RSA-2048 cryptography
- Prevent document forgery through cryptographic signatures and immutable ledger
- Enable independent verification by third parties (employers, partner institutions)
  without intermediaries
- Implement MIT Blockcerts standard for interoperable digital credentials

ARCHITECTURE:
- Frontend: React + Vite + Tailwind CSS + Framer Motion (responsive SPA)
- Backend: Node.js/Express + SQLite database acting as ledger node
- Cryptography: SHA-256 (hashing), RSA-2048 (digital signatures), JWT (authentication)
- Database: SQLite with tables for students, grades, certificates, blockchain logs,
  and revocation registry
- Authentication: JWT-based role system (admin, teacher, student roles)

CORE COMPONENTS:
1. Admin Dashboard - Grade management, student management, certificate issuance, revocation
2. Public Lookup - Student record retrieval with blockchain verification
3. Universal Verifier - Independent verification of downloaded JSON credentials
4. Blockchain Explorer - Transparent ledger viewing with block/transaction visualization
5. Key Management - RSA keypair generation and cryptographic operations

================================================================================
2. MAIN COMPONENTS & INTERACTIONS

DATA FLOW:
Admin Input → Backend Hash + Sign → SQLite DB + Blockchain Log

4. POTENTIAL BUGS, ERRORS & SECURITY VULNERABILITIES
================================================================================

🔴 CRITICAL (Production Blockers):

1. HARDCODED SECRET KEY
   Location: backend/server.js, line 16
   Code: const SECRET_KEY = "dummy_secret_for_demo";
   Impact: All JWT tokens can be forged. Any attacker can create admin tokens.
   Risk Level: CRITICAL
   Fix: Move to environment variables, use strong random keys for production.
   Time to Fix: 30 minutes

2. PLAIN TEXT PASSWORD STORAGE
   Location: backend/server.js, login endpoint
   Code: const user = await db.get('SELECT * FROM users WHERE username = ? AND password = ?', 
         [username, password]);
   Impact: Database breach exposes all passwords. No hashing (bcrypt/argon2).
   Risk Level: CRITICAL
   Fix: Hash passwords with bcrypt before storage.
   Time to Fix: 1 hour

3. LACK OF INPUT VALIDATION
   Location: Multiple endpoints (POST /api/students, /api/grades, /api/certificates)
   Impact: XSS, injection, malformed data - no sanitization of user inputs
   Risk Level: CRITICAL
   Fix: Implement comprehensive input validation and sanitization
   Time to Fix: 2-3 hours

4. UNENCRYPTED PRIVATE KEY ON DISK
   Location: backend/KeyService.js, lines 5-6
   Code: const PRIVATE_KEY_PATH = path.join(__dirname, 'private.pem');
   Impact: Compromised key file = compromised digital signatures
   Risk Level: CRITICAL
   Fix: Encrypt private key at rest using key derivation
   Time to Fix: 1.5 hours

5. NO HTTPS ENFORCEMENT
   Impact: All credentials transmitted over HTTP in demo mode
   Risk Level: CRITICAL
   Fix: Enforce HTTPS in production, use TLS certificates
   Time to Fix: 1 hour

🟡 MEDIUM (Must fix before beta):

6. RACE CONDITION IN GRADE CREATION
   Location: backend/server.js, POST /api/grades
   Code:
   const existing = await db.get('SELECT id FROM grades WHERE ...');
   if (existing) { await db.run('UPDATE ...'); }
   else { await db.run('INSERT ...'); }
   Problem: Two separate queries = race condition window
   Impact: Duplicate records possible if concurrent requests
   Fix: Use INSERT OR REPLACE or UPSERT clause
   Time to Fix: 30 minutes

7. N+1 QUERY PROBLEM IN BLOCKCHAIN VERIFICATION
   Location: backend/server.js, GET /api/grades
   Code:
   for (let g of grades) {
       const hashStatus = await bcm.verifyHash(dataString);  // N+1 queries!
   }
   Impact: Scales poorly; if 1000 grades, makes 1000 DB queries
   Fix: Pre-join blockchain_logs and revocations tables
   Time to Fix: 2 hours

8. NO DATABASE INDEXES
   Location: backend/db.js
   Impact: Lookups by student_id are full table scans (O(n))
   Fix: CREATE INDEX idx_grades_student_id ON grades(student_id);
   Time to Fix: 30 minutes

9. NO CORS VALIDATION
   Location: backend/server.js, line 9
   Code: app.use(cors());  // Allows ANY origin
   Impact: Cross-site attacks possible
   Fix: cors({ origin: process.env.ALLOWED_ORIGINS })
   Time to Fix: 30 minutes

10. MISSING ERROR HANDLING IN ASYNC ROUTES
    Location: Multiple POST endpoints (/api/grades, /api/certificates)
    Impact: Unhandled promise rejections can crash server
    Fix: Wrap all async operations in try-catch blocks
    Time to Fix: 1-2 hours

11. INEFFICIENT PRIVATE KEY I/O
    Location: backend/KeyService.js
    Code: fs.readFileSync(PRIVATE_KEY_PATH, 'utf8') called on every signature
    Impact: Disk I/O on every operation = performance bottleneck
    Fix: Load keys into memory at startup, cache in service
    Time to Fix: 30 minutes

12. MISSING REVOCATION TIMESTAMP HANDLING
    Location: backend/server.js, verify endpoint
    Impact: Revocation record has timestamp but not returned in lookup
    Fix: Include full revocation details in response
    Time to Fix: 30 minutes

🟠 LOW (Polish for v1.0):

13. No audit logging - No record of who accessed/modified student records
14. No data encryption at rest - SQLite database unencrypted
15. Frontend hardcoded URLs - Not configurable per environment
16. No pagination - Returns all students/grades at once (memory bloat)
17. QR code size limit - Large transcripts can't be encoded in QR (~2.9KB limit)
18. Frontend re-renders - All data re-fetched on each tab switch
19. No automated backups - Data loss risk
20. No rate limiting - Brute force attacks possible
21. Weak GPA calculation documentation - tx1-tx5 weight unclear

const bcrypt = require('bcrypt');

async function hashPassword(password) {
    return await bcrypt.hash(password, 10);

async function verifyPassword(inputPassword, hashedPassword) {
    return await bcrypt.compare(inputPassword, hashedPassword);
}

// In server.js login endpoint
app.post('/api/auth/login', async (req, res) => {
    const { username, password } = req.body;
    const user = await db.get('SELECT * FROM users WHERE username = ?', [username]);
    if (!user) return res.status(401).json({ error: "Invalid credentials" });
    
    const isValid = await verifyPassword(password, user.password);
    if (!isValid) return res.status(401).json({ error: "Invalid credentials" });
    
    const token = jwt.sign({ id: user.id, username: user.username, role: user.role }, SECRET_KEY);
    res.json({ token, role: user.role, username: user.username });
});

EXAMPLE 2: Input Validation

function validateStudentId(studentId) {
    return /^[A-Z0-9-]{3,20}$/.test(studentId.trim().toUpperCase());
}

function validateEmail(email) {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
}

function sanitizeString(str) {
    return String(str).trim().replace(/[<>\"']/g, '');
}

app.post('/api/students', authenticate, requireAdmin, async (req, res) => {
    let { student_id, name, email, class_name } = req.body;
    
    // Validate
    if (!validateStudentId(student_id)) 
        return res.status(400).json({ error: "Invalid student ID format" });
    if (!validateEmail(email)) 
        return res.status(400).json({ error: "Invalid email format" });
    if (!name?.trim()) 
        return res.status(400).json({ error: "Name is required" });
    
    // Sanitize
    student_id = student_id.toUpperCase();
    name = sanitizeString(name);
    class_name = sanitizeString(class_name);
    
    try {
        await db.run(
            'INSERT INTO students (student_id, name, email, class_name) VALUES (?, ?, ?, ?)', 
            [student_id, name, email, class_name]
        );
        res.json({ message: "Student added successfully" });
    } catch (err) {
        res.status(400).json({ error: "Duplicate student ID or invalid data" });
    }
});

EXAMPLE 3: Database Indexes

async function initDb() {
    const db = await open({
        filename: path.join(__dirname, 'database.sqlite'),
        driver: sqlite3.Database
    });

    // Create tables first...
    await db.exec(`
        CREATE TABLE IF NOT EXISTS students (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            student_id TEXT UNIQUE,
            name TEXT,
            email TEXT,
            class_name TEXT
        );
        
        CREATE TABLE IF NOT EXISTS grades (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            student_id TEXT,
            subject TEXT,
            tx1 REAL, tx2 REAL, tx3 REAL, tx4 REAL, tx5 REAL,
            gk REAL, ck REAL,
            score REAL,
            semester TEXT,
            hash TEXT,
            signature TEXT,
            FOREIGN KEY (student_id) REFERENCES students(student_id)
        );
        
        -- Add indexes for frequently queried columns
        CREATE INDEX IF NOT EXISTS idx_grades_student_id ON grades(student_id);
        CREATE INDEX IF NOT EXISTS idx_grades_hash ON grades(hash);
        CREATE INDEX IF NOT EXISTS idx_blockchain_logs_hash ON blockchain_logs(data_hash);
        CREATE INDEX IF NOT EXISTS idx_revocations_hash ON revocations(data_hash);
    `);

    return db;
}

EXAMPLE 5: Fix Race Condition with UPSERT

app.post('/api/grades', authenticate, requireStaff, async (req, res) => {
    const { student_id, subject, semester, tx1, tx2, tx3, tx4, tx5, gk, ck } = req.body;
    
    const score = calculateGPA({ tx1, tx2, tx3, tx4, tx5, gk, ck });
    const dataString = getGradeDataString({ student_id, subject, semester, tx1, tx2, tx3, tx4, tx5, gk, ck });
    const hash = await bcm.saveHash(dataString);
    const signature = KeyService.signData(dataString);
    
    try {
        // Atomic UPSERT operation - no race condition
        await db.run(`
            INSERT INTO grades (student_id, subject, semester, score, hash, signature, tx1, tx2, tx3, tx4, tx5, gk, ck)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
            ON CONFLICT(student_id, subject, semester) DO UPDATE SET
                score=excluded.score,
                hash=excluded.hash,
                signature=excluded.signature,
                tx1=excluded.tx1,
                tx2=excluded.tx2,
                tx3=excluded.tx3,
                tx4=excluded.tx4,
                tx5=excluded.tx5,
                gk=excluded.gk,
                ck=excluded.ck
        `, [student_id, subject, semester, score, hash, signature, tx1, tx2, tx3, tx4, tx5, gk, ck]);
        
        res.json({ message: "Grade added/updated and saved to Blockchain", hash, calculatedScore: score });
    } catch (err) {
        console.error('Grade creation error:', err);
        res.status(500).json({ error: 'Error saving grade' });
    }
});

async function initDb() {

        CREATE TABLE IF NOT EXISTS audit_logs (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            action TEXT,
            user_id INTEGER,
            user_name TEXT,
            resource_type TEXT,
            resource_id TEXT,
            changes TEXT,
            ip_address TEXT,

        );
        
        CREATE INDEX IF NOT EXISTS idx_audit_logs_user_id ON audit_logs(user_id);
        CREATE INDEX IF NOT EXISTS idx_audit_logs_timestamp ON audit_logs(timestamp);

async function logAudit(db, action, userId, userName, resourceType, resourceId, changes, ipAddress) {
    await db.run(
        `INSERT INTO audit_logs (action, user_id, user_name, resource_type, resource_id, changes, ip_address)
         VALUES (?, ?, ?, ?, ?, ?, ?)`,
        [action, userId, userName, resourceType, resourceId, JSON.stringify(changes), ipAddress]
    );
}
